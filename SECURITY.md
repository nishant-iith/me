# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it by emailing nishant.iith@gmail.com.

Please include:
- A description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Suggested fix (if any)

I will respond within 48 hours and work to address the issue promptly.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |
| Older   | ❌        |

## Security Best Practices

This project implements:
- Content Security Policy (CSP) headers
- Strict CORS configuration
- Rate limiting on API endpoints
- Input validation and sanitization
- Environment variables for sensitive data

## Dependencies

Dependencies are regularly updated. Run `npm audit` to check for vulnerabilities.
